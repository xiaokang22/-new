# 楼邦建材业绩报表系统 - 项目交接文档 v2

## 1. 项目目标
为建材店店长打造的 Web 端业绩管理系统，自动区分"到店购买"与"业务员推销"渠道，支持会员管理，生成日报/月报/年报，并确保数据永不丢失。

## 3. 已完成的功能

### 数据录入页（/entry）
- 选择日期录入销售记录
- 支持"到店购买"和"业务员推销"两种渠道
- 会员名字字段（用于会员业绩查询）
- 当日记录列表 + 汇总统计
- 删除记录（带二次确认）

### 报表页（/reports）
- **日报**：选日期 → 当日汇总 + 业务员业绩 + 销售明细列表（可删除）
- **月报**：选月份 → 该月总业绩 + 每个业务员该月业绩
- **年报**：选年份 → 该年总业绩 + 每个业务员该年业绩
- Excel 导出

### 业务员管理页（/salespersons）
- 增删改业务员
- 删除前检查是否有关联销售记录

### 会员业绩汇总页（/member）
- 选月份 + 输入会员名字 → 该会员本月每笔订单 + 总业绩

### 数据安全
- 每次写入/删除自动备份到 GitHub 私有仓库
- 启动时自动检测：如果数据库为空，从 GitHub 恢复
- 备份改为异步非阻塞（删除/新增操作秒回）

### Supabase 建表 SQL
```sql
-- 业务员表
CREATE TABLE salespersons (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    phone TEXT,
    position TEXT,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 销售记录表
CREATE TABLE sales (
    id SERIAL PRIMARY KEY,
    date TEXT NOT NULL,
    channel TEXT NOT NULL CHECK(channel IN ('store', 'salesperson')),
    salesperson_id INTEGER REFERENCES salespersons(id),
    amount NUMERIC NOT NULL CHECK(amount > 0),
    note TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 关闭 RLS（简化访问控制）
ALTER TABLE salespersons ENABLE ROW LEVEL SECURITY;
ALTER TABLE sales ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all" ON salespersons FOR ALL USING (true);
CREATE POLICY "Allow all" ON sales FOR ALL USING (true);
```
