# 楼邦建材业绩报表系统 - 项目交接文档 v3

## 1. 项目目标
为建材店店长打造的 Web 端业绩管理系统，自动区分"到店购买"与"业务员推销"渠道，支持退款、会员管理，生成日报/月报/年报，数据存储在云端永不丢失。

## 3. 已完成的功能

### 数据录入页（/entry）
- 选择日期录入销售记录
- **退款开关**：开启后，选择渠道（到店/推销）表示退款来源，金额为退款金额
- 业务员选择（推销时必选）
- 会员名字字段（用于会员业绩查询，退款时也可填写）
- 当日记录列表（退款记录红色高亮显示）+ 汇总统计（总业绩/到店/推销/退款）
- 删除记录（带二次确认）

### 报表页（/reports）
- **日报**：选日期 → 汇总 + 业务员业绩（已扣除退款）+ 销售明细（可删除）
- **月报**：选月份 → 总业绩 + 每日趋势 + 业务员月度业绩
- **年报**：选年份 → 总业绩 + 每月趋势 + 业务员年度业绩
- 退款金额在汇总卡片中单独显示（红色）
- Excel 导出（前端 xlsx 生成，退款金额显示为负数）

### 业务员管理页（/salespersons）
- 增删改业务员（姓名、手机、职位、启用/禁用）
- 删除前检查是否有关联销售记录

### 会员业绩汇总页（/member）
- 选月份 + 输入会员名字 → 该会员本月每笔订单（含退款）+ 净业绩

## 4. 关键技术栈
- **前端**：Vue 3 + Vite 5 + Element Plus + Vue Router 4
- **数据库**：Supabase（PostgreSQL），项目名 `loubang-建材业绩`
- **部署**：Cloudflare Pages（国内可直连）+ Vercel（备用，国内需翻墙）
- **Excel 导出**：SheetJS（xlsx 库，前端生成）
- **无后端服务器**：前端直接调用 Supabase SDK

## 6. 数据库结构（Supabase）

### salespersons 表
| 字段 | 类型 | 说明 |
|---|---|---|
| id | SERIAL PK | 自增主键 |
| name | TEXT | 姓名（必填） |
| phone | TEXT | 手机号 |
| position | TEXT | 职位 |
| is_active | BOOLEAN | 是否启用（默认 true） |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |

### sales 表
| 字段 | 类型 | 说明 |
|---|---|---|
| id | SERIAL PK | 自增主键 |
| date | TEXT | 日期（YYYY-MM-DD） |
| channel | TEXT | 渠道：store（到店）/ salesperson（推销） |
| salesperson_id | INTEGER FK | 业务员 ID（推销时必填） |
| amount | NUMERIC | 金额（>0） |
| note | TEXT | 会员名字 / 退款原因 |
| is_refund | BOOLEAN | 是否退款（默认 false） |
| created_at | TIMESTAMPTZ | 创建时间 |

## 7. 已知问题
- **Vercel 国内无法直连**：需要用 Cloudflare Pages 部署
- **Supabase 免费版限制**：500MB 数据库、50,000 行数据、2GB 带宽/月
- **退款与渠道关联**：退款必须选择渠道（到店/推销），用于正确扣减对应渠道业绩
- **业务员业绩计算**：日报从 API 获取 salesperson_data（API 内部处理退款扣减），月报/年报同理
